How it works

1
Parse GPX — reads track points, route points, or waypoints (in that priority order).
2
Reduce waypoints — Google Maps supports ~9-10 waypoints for reliable app hand-off. Intermediate points are evenly sampled while start and end are always preserved.
3
Build deep link — generates a google.com/maps/dir/ URL that opens native turn-by-turn navigation on mobile.
4
Open on phone — copy the link or scan a QR code, then open it on your phone to start navigation immediately.
Everything runs in your browser — no files are uploaded to any server.
Python CLI version also available.
